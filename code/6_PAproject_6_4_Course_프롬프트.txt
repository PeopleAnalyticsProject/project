다음 조건을 모두 만족하는 Python 추천 시스템 코드를 작성해줘. 

 목적
직원에게 맞춤 교육과정을 추천하는 시스템을 단일 .py 파일로 구현.

 환경
- 언어: Python 3.x
- 실행환경: Windows 로컬 PC + PyCharm IDE
- 별도 서버/클라우드 없이 로컬에서 단일 .py 파일로 실행
- 데이터: 로컬 엑셀 파일 경로를 코드 상단 상수(EXCEL_PATH)로 지정
- 필요 패키지: pandas, numpy, scikit-learn, scipy, openpyxl
- 코드 상단 주석에 pip install 명령어 포함

 데이터 구조 (엑셀 파일, 시트 5개)
| 시트명           | 주요 컬럼                                                                    |
|-----------------|-----------------------------------------------------------------------------|
| courses         | course_id, name, category, level                                            |
| employees       | emp_id, dept, grade, grade_num, tenure, ai_literacy, data_skill, leadership |
| ratings_train   | emp_id, course_id, rating                                                   |
| ratings_test    | emp_id, course_id, rating                                                   |
| recommend_target| emp_id + employees와 동일 컬럼 (추천 대상 직원)                               |

 구현할 모듈 (클래스 단위로 분리)

 1. 데이터 로드 함수 (load_excel, build_rating_matrix)
- 5개 시트 존재 여부 검증 후 로드
- 직원 × 교육과정 평점 행렬 생성 (미수강=0)

 2. CollaborativeFilter 클래스 (협업 필터링)
- scipy.sparse.linalg.svds로 SVD 행렬 분해
- 사용자 평균 기반 행 중심화(row centering) 적용
- recommend(emp_id, top_n, exclude_rated) 메서드 포함
- exclude_rated=True일 때 rating_matrix에서 rating > 0인
  과목을 score에서 -inf로 마스킹하여 완전히 제거

 3. ContentFilter 클래스 (콘텐츠 필터링)
- 직원 프로필 벡터: grade_num, tenure, ai_literacy, data_skill,
  leadership + 카테고리 더미 (9개 카테고리 원핫)
- 교육과정 벡터: level, category 원핫인코딩
- 코사인 유사도로 점수 산출
- recommend(emp_id, top_n, exclude_rated) 메서드 포함
- exclude_rated=True일 때 rating_matrix에서 rating > 0인
  과목을 결과에서 완전히 제거 (필터링 후 top_n 반환)

 4. HybridRecommender 클래스 (하이브리드 추천)
★ 아래 3가지 로직을 반드시 정확하게 구현할 것 ★

  [4-1] 점수 정규화 (MinMax Scaling 필수)
  - CF 점수와 CB 점수는 스케일이 다르므로 (CF: 1~5점대, CB: 0~1점대)
    단순 가중합 금지.
  - 반드시 sklearn.preprocessing.MinMaxScaler를 사용하여
    두 점수를 각각 독립적으로 0~1 범위로 정규화한 뒤 가중합 계산.
  - 공식: hybrid_score = alpha * cf_score_normalized
                       + (1 - alpha) * cb_score_normalized

  [4-2] 동적 가중치 (Dynamic Alpha) 필수
  - 직원별 수강 이력 수에 따라 alpha를 동적으로 결정:
      num_rated = (rating_matrix.loc[emp_id] > 0).sum()
      if num_rated <= 2:
          effective_alpha = 0.2    CB 80% 반영 (프로필 기반 개인화)
      else:
          effective_alpha = ALPHA   기본값 (설정된 alpha 사용)
  - 수강 이력이 적은 직원에게 인기 과목만 추천되는
    cold-start 문제를 방지하기 위함.

  [4-3] 미수강 과목 필터링 (Exclude Rated 정교화)
  - exclude_rated=True일 때 CF, CB, Hybrid 세 모델 모두에서
    rating_matrix 기준 rating > 0인 과목을 점수 계산 전에 제거.
  - 점수 계산 후 사후 필터링이 아닌,
    점수 풀(pool) 자체에서 사전 제거(pre-filtering) 방식으로 구현.
  - recommend(emp_id, top_n, exclude_rated) 메서드 포함

 5. 평가 함수
- evaluate(): Precision@K, Recall@K 계산
- rmse_score(): CF 평점 예측 RMSE 계산

 6. main() 함수
- 모델 3종(CF, CB, 하이브리드) 평가 결과 콘솔 출력
- recommend_target 시트의 직원별 Top-N 추천을
  3가지 방법 각각으로 콘솔 출력
- 하이브리드 Top-3 결과를 원본 엑셀과 같은 폴더에
  recommendations.xlsx로 저장

 하이퍼파라미터 (코드 최상단에 상수로 분리)
EXCEL_PATH = r"C:\Users\user\Desktop\6_PAproject_6_4_course.xlsx"
TOP_N      = 5       추천 과목 수
ALPHA      = 0.6     하이브리드 기본 CF 가중치 (0~1), 동적 조정됨
N_FACTORS  = 5       SVD 잠재 요인 수
